Инициализируем проект, появляется файл package.json

1) npm init


устанавливаем WebPack, где -D зависмости только в разработке => devDependencies. WebPack - кореневой функционал, webpack cli - пакет отвечает за команды в консоли

2) npm install -D webpack webpack-cli


3) создаем файл webpack.config.js, где mode - режим минифицирован или нет, entry - от куда брать код, output - куда вставлять

Заработали команды в консоли только после установки данного кода

4) npm install -g --save-dev webpack-cli

команда webpack запускает сборку


Устанавливаем плагин для подключение динамических url в index.html

5) npm install -D html-webpack-plugin


удаляем ненужные файлы после нового deploy

6) npm install -D clean-webpack-plugin


плагины для отображения css файлов

7) npm i -D style-loader css-loader


плагины для отображения картинок и других файлов

8) npm install -D file-loader


9) npm install normalize.css


9) npm i -D xml-loader

9) npm i -D html-loader


пакет для чтения csv

9) npm i -D papaparse

resolve: { extensions: ['js']} - для чтения файлов без их расширения
optimization: {splitChunks: {chunks: "all"}}- нужно чтобы библиотеки не подгружались по несколько раз в разных компонентах

пакет для автоматической пересборки приложения

9) npm install -D webpack-dev-server


пакет переброски данных

10) npm i -D copy-webpack-plugin


пакет для css создает файл

11) npm install --save-dev mini-css-extract-plugin


12) npm i -D cross-env (1:50 - время)


пакет оптимизируем js

13) npm install terser-webpack-plugin --save-dev


пакет оптимизируем css

14)npm i --save-dev optimize-css-assets-webpack-plugin

npm install




запуск сборки
npm run dev
